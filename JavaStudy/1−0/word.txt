IT：インターネットなどの通信とコンピュータとを駆使する情報技術そのもの
ICT：通信技術を使って人と人、人とインターネットが繋がる技術の指す。
loT:モノが自動的にインターネットに繋がる技術（外出先での利用等）
OS:システム全体を管理し、さまざまなアプリケーションソフトを動かすための最も基本的なソフトウェアのこと
Windows:Microsoftが提供しているOSのこと
macOS:Appleが開発・販売するMacのオペレーティングシステム
UNIX:AT&T社ベル研究所で開発が始まったオペレーティングシステム（OS）。また、その流れを汲むOSの総称。
Linux:システム開発、特にOS開発やサーバ構築で役立つOS
マークアップ／フロントエンド:文書構造（テキスト）や視覚表現（装飾）などの情報を、コンピューターが正しく認識できるように、タイトルや見出しなどの各構成要素に「タグ」と呼ばれる識別のための目印を使い、意味付けを行っていくこと
HTML：web制作においてコンピュータが理解できる、マークアップをするための言葉
CSS：ウェブページのスタイルを指定するための言語の一つ
JavaScript：ユーザー側のWebブラウザと、Webサイトまたはウェブサービスの相互間のやりとりを、円滑にするために使われている原語
jQuery：ウェブブラウザ用のJavaScriptコードをより容易に記述できるようにするために設計されたJavaScriptライブラリ
Java SE：プログラミング言語「Java」におけるAPIなどの機能セットの標準の一つで、基本的で汎用的な機能をまとめたもの。
サーバーサイド：コンピュータネットワーク上のクライアント-サーバ関係において、サーバが行う操作
.NET:ネットワーク上でアプリケーションを構築する基盤システムのこと。
・Java
Java SE: パソコン上で動作するゲームや便利なソフトを作ることができる。Java言語でプログラミングを行う際に最低限必要な機能をまとめたもの
Java EE（Jakarta EE）:JavaSE+拡張機能(SEにサーバーサイドの機能を追加したもの)
Perl: ラリー・ウォールによって開発されたプログラミング言語(バージョンがアップデートされても、古いバージョンのコードが扱える)
PHP: 掲示板等の動的なWebページの生成が必要なさまざまなアプリケーションで使われる。
Ruby: ラリー・ウォールによって開発されたプログラミング言語(文法が分かりやすくシンプルなコードが書けること)
■ サーバーソフト
Apache: 大量の動的コンテンツの処理に向いており 動画を中心としたWebサイトを運営するときに使われる。
nginx:フリーかつオープンなWebサーバ用のソフトウェアでサーバ上のファイルを高速に配信するように設計されている。
IIS:Microsoft社の「無料」で利用できるWebサーバーソフトウェアでWindowsに標準搭載されているもの
■ SQL
SQL:DBMS（データベース管理システム）上でデータやデータベースを制御するための言語
DB2:SQLを初めて採用したデータベース管理システム（ リレーショナル【関係付け】型データベース管理システム（RDBMS）です）。
MySQL:オープンソースのリレーショナルデータベース管理システム 
Maria DB:オープンソースのリレーショナルデータベース管理システム 
Oracle DB:堅牢さを持つリレーショナルデータベース(行と列からなる表形式の構造をもつ)
PostgreSQL: Webシステムで使われるデータベース
SQLite: オープンソースのリレーショナルデータベースの一種。他のソフトウェアに組み込んで利用することを想定した軽量な仕様が特徴。
SQL Server: Windows OSで動作させることができるリレーショナルデータベースの一種
CRUD: リレーショナルデータベース上で、どのデータに対してどのプロセスが生成(Create)、読み込み(Read)、更新(Update) 、削除(Delete)されるかを表す表形式のマトリックス
トランザクション: 主にデータベースなどにおいて使われる用語で、「複数の処理を一つの処理としてまとめたもの」のこと
コミット:管理システム内のすべての変更を他のユーザーに反映する
ロールバック:システム障害が起きた時に以前に正常に稼働していたある時点の状態に戻して復旧を試みること
排他制御（排他ロック）: 割り込み禁止機構(あるプロセスに資源を独占的に利用させている間は、他のプロセスが利用できないようにする事で整合性を保つ処理の事)
楽観ロック: データそのものに対してロックは行わずに、更新対象のデータがデータ取得時と同じ状態であることを確認してから更新することで、データの整合性を保証する方式(滅多に他者との同時更新は起きない前提)
悲観ロック: データを取得する際にロックをかけることで、他のトランザクションから更新されないようにする方式。(他者が同じデータに頻繁に変更を加えるであろう、という悲観的な前提)
実行計画: SQLserverがユーザーが発行したSQLを解析し決定した実データに対して、どのようにアクセス・計算を行うかが記載されている計画書
チューニング: 設定や構成を調整し、目標の状態に近づけたり、性能を最大限引き出したりする作業を指す
パフォーマンス:機器の処理性能や実行速度、通信回線・ネットワークなどの伝送速度・容量などのことを指す
スロークエリ: データベース管理システム（DBMS）に対する問い合わせ（クエリ）のうち、一定の基準に照らして遅い、時間がかかっているもの。ボトルネック箇所
クエリログ: クライアントから実行された全ての SQL クエリを出力してくれるログのこと
IDE（統合開発環境）: 開発に必要なソフトウェアを一つに組み合わせ、同じ操作画面から統一的な操作法で利用できるようにしたソフトウェアパッケージ
Eclipse: コンピュータプログラミングにおいて使用される統合開発環境
NetBeans: プロジェクト管理機能や入力支援機能を備えたオープンソースのIDE
SpringToolSuite: Eclipseと同様にSpringベースのアプリケーション開発を行うために提供される開発ツール
DBツール: 1つのデータベース・ツールですべてのデータベース管理タスクを実行することはできない
A5:SQL Mk-2: 複雑化するデータベース開発を支援するために開発されたフリーの汎用SQL開発ツール
pgAdmin: PostgreSQLの管理ツールの一つで、ネットワークを通じて遠隔のデータベースを表や図で管理することができる
DBeaver: ほぼ全てのDBに対応したSQLクライアントツール
MySQL Workbench: MySQL公式で配布されているデータベース管理ツール
Sequel Pro: データベースをExcelのような表で確認・編集をすることができるMac専用のアプリ
TeamSQL: 完全無料を謳っているDBをGUIで管理する為のソフトウェアのひとつ
■ 肩書
PM:プロジェクトマネージャー(計画、運営管理等全工程にかかわる)
PL: いずれかのフェーズのリーダーとしてプロジェクトメンバーを管理、統括
PG（PE）: プログラミング言語やソフトウェア開発ツールなどを用いてコンピュータプログラムを作成・修正する技術者
SE: ソフトウェアの開発に携わる技術者のうち、プログラミング以外の業務を担当する者
デザイナー: WebページやDTPに関するレイアウトや、素材の制作を担当
エンジニア
フロントエンド・エンジニア: 利用者への表示や操作の受け付けを行う部分を開発する
サーバーサイド・エンジニア: サーバ側でおこなう処理の設計・プログラムの開発、サーバで扱うデータのシステム構築・管理・保守・改善、バグの修正を担当
バックエンド・エンジニア: フロントエンド側からデータや指示を受け付け、計算や変換などの処理を行ったり、ストレージやデータベースなどでデータの保存や読み出しを行う部分の開発を行う
■ UI（ユーザーインターフェース）
UI
CUI: コンピュータやソフトウェアが利用者に情報を提示したり操作を受け付けたりする方法の一つで、すべてのやり取りを文字によって行う方式
GUI: コンピュータへ出す命令や指示等を、ユーザが画面上で視覚的に捉えて行動を指定できるものの為直感的に操作可能
■ ファイル／ディレクトリ
ファイル: データの管理単位の一つで、データを記録する際に利用者やOSから見て最小の記録単位となるデータのまとまり。
拡張子: ファイルの種類や形式を表すために利用者やソフトウェアによって付与されるもの(.の右側)
ディレクトリ
ルート・ディレクトリ:ストレージ内のファイルにおいて最上位のある唯一のディレクトリのこと(ツリー構造)
カレント・ディレクトリ: 実行中のソフトウェアなどがストレージ内の現在位置を指し示すディレクトリの事
パス
絶対パス: ファイルなどの所在を書き表す方法の一つで、階層構造の頂点からの位置関係を記述する方式
相対パス: 起点となる位置(現在位置)から目的の位置までの道筋にある要素を順に並べて記述する。
■ ネットワーク
LAN:建物内の限定的なネットワーク
WAN: 遠く離れたエリアとつながったネットワーク
クライアント: 他のコンピュータやソフトウェアから機能や情報の提供を受けるコンピュータやソフトウェアのこと
サーバー: ネットワーク上でクライアントから要求や指示を受け、情報や処理結果を返す役割を持つコンピュータやソフトウェアのこと
■ 各種プロトコル
HTTP: WebサーバとWebクライアントの間でデータの送受信を行うために用いられるプロトコル
FTP: ファイル転送プロトコル。WEBページを公開するときに使用され、作成したHTMLなどのファイルをサーバーにアップロードしたり、更新したりする際に利用される。
HTTPS: WebサーバとWebブラウザの間の通信が暗号化されていることを意味し、通信経路上での盗聴や改竄、第三者によるなりすましを防止する役割
SSL: ウェブブラウザとウェブサーバ間でのデータの通信を暗号化し、送受信させる仕組み
SSH: ネットワークに接続された機器を遠隔操作し、管理するための手段
ポート、ポート番号:通信手段の一つで同じコンピュータ内で動作する複数のソフトウェアのどれが通信するかを指定する。
HTTP、FTP、SSHの各ポート番号: TCP 80、TCP 20 or TCP 21、TCP 22
ファイアウォール: 通信のあて先の情報をもとにして、不正が疑われる通信を遮断する仕組み
TCP: 相手との1対1で信頼性の高い通信手段
■ IPアドレス
IPアドレス(通信相手を識別する番号の事)
グローバルIPアドレス: インターネットを利用するときに割り振られる
プライベートIPアドレス: LANなどの企業内ネットワークに存在する端末が、自由に使用できるIPアドレスのこと。
■ ドメイン
ドメイン: インターネット上の住所表示の様なもの。電子メールを送ったり、ウェブサイトを閲覧する際に相手を特定
DNS: インターネット上で ドメイン名 を管理・運用するために開発されたシステム
VPN: インターネット回線を利用して作られる仮想のプライベートネットワークのこと
ブリッジ: ネットワーク中のデータの中にあるMACアドレスを見て、行き先を振り分けてくれる機器のこと
プロキシサーバー: Webブラウザの代わりにホームページにアクセスしてくれるコンピュータのこと
■ 電子メール
SMTP: メールを相手のメールサーバーまで届ける仕組み
POP: 自動的にメールサーバーにアクセスしてメールをダウンロードする仕組み
IMAP: 受信メールをサーバ上で管理し、メールソフトに表示させる受信方式(オフラインでも最後に受信していたものが見れる)
■ セッションとクッキー
セッション:通信開始から終了まで
クッキー: Webサーバーがクライアントコンピュータに預けておく小さなファイルのこと
キャッシュ:読み出したデータの内直近に読み込んだものや使用頻度が高いものを高速な記憶装置に複製しておくこと
■ DB（データベース）について
インデックス: 書籍の巻末の索引(必要な情報を素早く検索)
■ テストについて
フロー: 情報を追加して不正なパターンを探索する事
■ チケット管理ツール
タスク:コンピュータがやるべき処理単位
■ バージョン管理
Git: ファイルのバージョン管理が簡単にできるツール(変更、更新)
SVN: ファイルのバージョン変更履歴を管理してくれるシステムのひとつ
デグレ:デグレード(前の状態の方が良かった事を指す)
マージツール:複数のものを統合、結合する為のツール
